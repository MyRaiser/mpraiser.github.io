无符号数

我们知道一个数字电路的输出具有两种状态：000或者111，称为一个bit。那么如果有nnn个bit，则可以表示的状态总数有2n2^n2n个。一个最简单的思路是直接将各比特的值当作二进制数各位上的数码，那么，一个比特序列[X]u=xn−1⋯x1x0[X]_u = x_{n-1}\cdots x_1 x_0[X]u​=xn−1​⋯x1​x0​对应的实际数值为

X=∑i=0n−1xi⋅2iX = \sum_{i=0}^{n-1} x_i \cdot 2^iX=i=0∑n−1​xi​⋅2i

表示范围是

X:0∼2n−1[X]u:00⋯0⏟n∼11⋯1⏟n\begin{aligned} X &: &0 &\sim 2^n - 1 \\ [X]_u &: &\underbrace{00 \cdots 0}_n &\sim \underbrace{11 \cdots 1}_n \end{aligned}X[X]u​​::​0n00⋯0​​​∼2n−1∼n11⋯1​​​

无符号（unsigned）数就是这么表示的，无符号数只能表示非负数，nnn位无符号数表示[0,2n)[0, 2^n)[0,2n)之间的数。

无符号数加法器

我们首先实现无符号数的加法器。特别注意，由于只有nnn位，实际上实现的是

(a+b)mod  2n(a + b) \mod 2^n(a+b)mod2n

因此我们说无符号数加法器实现的其实是模意义上的“+++”。无符号数的表示集合与无符号数加法构成了一个阿贝尔群⟨Nu,+u⟩\langle N_u, +_u\rangle⟨Nu​,+u​⟩。

有符号数

对于实际应用中光零与正数是不够的，我们还需要考虑的问题有：

  1. 如何表示负数？
  2. 如何实现减法？

原码

如果首先考虑表示负数的问题，要表达负数怎么办呢？

最naive的做法是对于nnn位有符号数，可以选取111个符号位用来充当正负号，剩余n−1n-1n−1位使用无符号数的方式表示数的绝对值。

表示范围是(−2n−1,2n−1)(-2^{n-1}, 2^{n-1})(−2n−1,2n−1)：

X:0∼2n−1−1,−0∼−(2n−1−1)[X]t:0_00⋯0⏟n−1∼0_11⋯1⏟n−1,1_00⋯0⏟n−1∼1_11⋯1⏟n−1\begin{aligned} X &: &0 &\sim &2^{n-1} - 1 &, &-0 &\sim &-(2^{n-1} - 1) \\ [X]_t &: &0\_\underbrace{00 \cdots 0}_{n-1} &\sim &0\_\underbrace{11 \cdots 1}_{n-1} &, &1\_\underbrace{00 \cdots 0}_{n-1} &\sim &1\_\underbrace{11 \cdots 1}_{n-1} \end{aligned}X[X]t​​::​00_n−100⋯0​​​∼∼​2n−1−10_n−111⋯1​​​,,​−01_n−100⋯0​​​∼∼​−(2n−1−1)1_n−111⋯1​​​

这种对于正负数的编码方式称作原码（true form）。

反码

现在该考虑怎么实现减法了。这个时候我们发现了大问题：原码的加减法实在是太复杂了。加法就很复杂了，还需要做一完全不一样的减法，能不能只用一套器件同时实现加减法？

我们知道，对于两个正数，减去一个数相当于加上其相反数（即加法逆元）：

a−b=a+(−b)a - b = a + (-b)a−b=a+(−b)

对于逆元有：

x+(−x)=0x + (-x) = 0x+(−x)=0

但是在使用+u+_u+u​的原码表示中，两个相反数之和不能满足

[X]t+u[−X]t=[0]t[X]_t +_u [-X]_t = [0]_t[X]t​+u​[−X]t​=[0]t​

比如4位下[1]t[1]_t[1]t​与[−1]t[-1]_t[−1]t​之和、[2]t[2]_t[2]t​与[−2]t[-2]_t[−2]t​之和为

0001+1001=10020010+1010=10200001 + 1001 = 1002 \\ 0010 + 1010 = 10200001+1001=10020010+1010=1020

并且这个问题也没有办法通过简单的对符号位处理得到正确答案。不过通过观察可以发现把负数的表示中除了符号位以外的部分取个反就可以形式上解决这个问题：

X:0∼2n−1−1,−(2n−1−1)∼−0[X]t:0_00⋯0⏟n−1∼0_11⋯1⏟n−1,1_00⋯0⏟n−1∼1_11⋯1⏟n−1\begin{aligned} X &: &0 &\sim &2^{n-1} - 1 &, &-(2^{n-1} - 1) &\sim &-0 \\ [X]_t &: &0\_\underbrace{00 \cdots 0}_{n-1} &\sim &0\_\underbrace{11 \cdots 1}_{n-1} &, &1\_\underbrace{00 \cdots 0}_{n-1} &\sim &1\_\underbrace{11 \cdots 1}_{n-1} \end{aligned}X[X]t​​::​00_n−100⋯0​​​∼∼​2n−1−10_n−111⋯1​​​,,​−(2n−1−1)1_n−100⋯0​​​∼∼​−01_n−111⋯1​​​

表示范围与原码相同，仍为(−2n−1,2n−1)(-2^{n-1}, 2^{n-1})(−2n−1,2n−1)；这种编码方式称为反码（one's complement）。

反码可以保证

[X]1c+u[−X]1c=11⋯1⏟n[X]_{1c} +_u [-X]_{1c} = \underbrace{11 \cdots 1}_n[X]1c​+u​[−X]1c​=n11⋯1​​

补码

原码与反码依然具有一些显而易见缺点：

  1. 000被以+0+0+0和−0-0−0的方式重复表达了两遍
  2. 反码虽然解决了逆元的问题，但是加法器本身还是很复杂，并且也不是+u+_u+u​

现在让我们重新审视加法。对于[0,2n)[0, 2^n)[0,2n)上的模加法，其加法逆元满足

−X={0,X=02n−X,X>0-X = \left\{\begin{aligned} 0, X = 0 \\ 2^n -X, X > 0 \end{aligned}\right.−X={0,X=02n−X,X>0​

前文提过无符号数⟨Nu,+u⟩\langle N_u, +_u\rangle⟨Nu​,+u​⟩中的无符号加法+u+_u+u​其实就是模加法，所以沿用+u+_u+u​来做有符号的加法是可行的，但是要做一些调整。我们仍然选取111位做符号位，剩余n−1n-1n−1位作如下处理：

  1. 如果是正数或者零，保持不变
  2. 如果是负数，取+u+_u+u​的逆元（这使得我们在有符号数中仍然能使用无符号的+u+_u+u​！）

dec:0∼2n−1−1,−2n−1∼−1bin:0_00⋯0⏟n−1∼0_11⋯1⏟n−1,1_00⋯0⏟n−1∼1_11⋯1⏟n−1\begin{aligned} \text{dec}&: &0 &\sim &2^{n-1} - 1 &, &-2^{n-1} &\sim &-1 \\ \text{bin}&: &0\_\underbrace{00 \cdots 0}_{n-1} &\sim &0\_\underbrace{11 \cdots 1}_{n-1} &, &1\_\underbrace{00 \cdots 0}_{n-1} &\sim &1\_\underbrace{11 \cdots 1}_{n-1} \end{aligned}decbin​::​00_n−100⋯0​​​∼∼​2n−1−10_n−111⋯1​​​,,​−2n−11_n−100⋯0​​​∼∼​−11_n−111⋯1​​​

表示范围为[−2n−1,2n−1)[-2^{n-1}, 2^{n-1})[−2n−1,2n−1)。这种编码方式称为补码（two's complement）。

对于符号位：

  1. 如果两数符号位一致，那么和的符号位保持相同。

  2. 如果两数符号位不一致，那么和是大的操作数减去小的操作数，加和结果的符号位和较大的操作数符号位保持一致。

再谈反码

反码在工程实践上可能出现的比补码更早。

对于反码，如果要求[−X]1c[-X]_{1c}[−X]1c​，做法是：

  1. 符号位不变
  2. 其他位取反

考虑一个正数XXX, [X]1c[X]_{1c}[X]1c​符号位以外的位为n−1n-1n−1位无符号数[Y]1c[Y]_{1c}[Y]1c​，[−X]1c[-X]_{1c}[−X]1c​对应的为[−Y]1c[-Y]_{1c}[−Y]1c​，我们知道：

[Y]1c+u[−Y]1c=11⋯1⏟n−1[Y]_{1c} +_u [-Y]_{1c} = \underbrace{1 1 \cdots 1}_{n - 1}[Y]1c​+u​[−Y]1c​=n−111⋯1​​

而11⋯1⏟n−1\underbrace{1 1 \cdots 1}_{n - 1}n−111⋯1​​实际上就是无符号数中的2n−1−12^{n-1} - 12n−1−1，因此有

[Y]1c+[−Y]1c=2n−1−1[Y]_{1c} + [-Y]_{1c} = 2^{n-1} - 1[Y]1c​+[−Y]1c​=2n−1−1

[−Y]1c=2n−1−1−[Y]1c[-Y]_{1c} = 2^{n-1} - 1 - [Y]_{1c}[−Y]1c​=2n−1−1−[Y]1c​

可以看到这个形式上与模加法的逆元[−Y]2c=2n−1−[Y]2c[-Y]_{2c} = 2^{n-1} - [Y]_{2c}[−Y]2c​=2n−1−[Y]2c​非常相像。

首先从计算形式上，负数的补码非符号位可以通过反码+1+1+1得到，有：

[−Y]2c=[−Y]1c+1=2n−1−[Y][-Y]_{2c} = [-Y]_{1c} + 1 = 2^{n-1} - [Y][−Y]2c​=[−Y]1c​+1=2n−1−[Y]

这提供了一个方便的得到+u+_u+u​逆元的方法。

反码具有与原码一样的问题：000被重复编码为+0+0+0、−0-0−0

从补码的设计思想上回过来看，这不仅仅是浪费了一个码位的问题，而是破坏了代数结构，使得加法器的设计比较困难。
